    Authentication

    To authorize, use this code:

    # With shell, you can just pass the correct header with each request
curl "https://api.give.io/v1/authenticate"
  -H "Authorization: <API_KEY>"

    // Javascript Examples Coming Soon

    Make sure to replace <API_KEY> with your valid API key.

    Give.io uses API keys to allow access to the API. During beta development, public access is not available.

    Give.io expects for the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: <API_KEY>

    You must replace <API_KEY> with your personal API key.

    Users

    Get All Users

    curl "https://api.give.io/v1/users"
  -H "Authorization: <API_KEY>"

    // Javascript Examples Coming Soon

    The above request returns a collection of JSON objects structured like this:

    [
  {
    "id": 1,
    "first_name": "",
    "last_name": "",
    "email": "",
    "avatar": "",
    "created_at": ""
  }
]

    This endpoint retrieves all users.

    HTTP Request

    GET https://api.give.io/v1/users

    Get a Specific User

    curl "https://api.give.io/v1/users/2"
  -H "Authorization: <API_KEY>"

    // javascript

    The above command returns JSON structured like this:

    {
  "id": 1,
  "first_name": "",
  "last_name": "",
  "email": "",
  "avatar": "",
  "created_at": ""
}

    This endpoint retrieves a specific user.

    HTTP Request

    GET https://api.give.io/v1/users/<ID>

    Delete a Specific User

    curl "https://api.give.io/v1/users/2"
  -X DELETE
  -H "Authorization: <API_KEY>"

    // Javscript

    The above command returns JSON structured like this:

    {
  "id": 1,
  "first_name": "",
  "last_name": "",
  "email": "",
  "avatar": "",
  "created_at": "",
  "deleted_at": ""
}

    This endpoint deletes a specific user.

    HTTP Request

    DELETE https://api.give.io/v1/users/<ID>

    This action is immediate and irreversible.
